John the Ripper

Advanced password cracking and hash analysis tool

Introduction to John the Ripper

John the Ripper is a fast password cracker, currently available for many flavors of Unix, macOS, Windows, DOS, BeOS, and OpenVMS. Its primary purpose is to detect weak Unix passwords.

Key Features

  • Multiple hash format support
  • Dictionary and brute force attacks
  • Rule-based password generation
  • Distributed cracking support
  • Custom character sets

Installation

John comes pre-installed with Kali Linux. For other systems:

# Ubuntu/Debian sudo apt-get install john # From source (Jumbo version) git clone https://github.com/magnumripper/JohnTheRipper.git cd JohnTheRipper/src ./configure && make # Check installation john --version

Basic Usage

Basic John the Ripper commands and operations:

# Basic password cracking john passwords.txt # Using wordlist john --wordlist=/usr/share/wordlists/rockyou.txt passwords.txt # Show cracked passwords john --show passwords.txt # Crack specific hash format john --format=md5 hashes.txt # Resume interrupted session john --restore

Supported Hash Formats

John supports numerous hash formats:

Common Formats

  • MD5 (md5)
  • SHA-1 (sha1)
  • SHA-256 (sha256)
  • SHA-512 (sha512)
  • NTLM (nt)
  • LM (lm)

System Formats

  • Unix crypt (crypt)
  • bcrypt (bcrypt)
  • scrypt (scrypt)
  • PBKDF2 (pbkdf2)
  • Argon2 (argon2)
# List all supported formats john --list=formats # Identify hash format john --list=format-details | grep -A5 "md5" # Format-specific cracking john --format=sha256 --wordlist=wordlist.txt hashes.txt

Attack Modes

Different attack strategies in John the Ripper:

Dictionary Attack

# Basic dictionary attack john --wordlist=passwords.txt hashes.txt # Multiple wordlists john --wordlist=list1.txt,list2.txt hashes.txt # External wordlist john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt

Brute Force Attack

# Incremental mode (brute force) john --incremental hashes.txt # Custom character set john --incremental=digits --min-length=4 --max-length=8 hashes.txt # ASCII brute force john --incremental=ascii hashes.txt

Rule-based Attack

# Using rules with wordlist john --wordlist=passwords.txt --rules hashes.txt # Specific rule set john --wordlist=passwords.txt --rules=best64 hashes.txt # Custom rules john --wordlist=passwords.txt --rules=custom hashes.txt

Advanced Features

Advanced John the Ripper techniques:

Performance Optimization

# Use multiple CPU cores john --fork=4 hashes.txt # GPU acceleration (if available) john --format=sha256-opencl hashes.txt # Memory usage optimization john --mem-file-size=1024 hashes.txt

Session Management

# Named session john --session=mysession hashes.txt # Restore specific session john --restore=mysession # Status of running session john --status=mysession

Hash Extraction

# Extract from /etc/shadow sudo unshadow /etc/passwd /etc/shadow > mypasswd.txt # Extract from Windows SAM samdump2 system sam > hashes.txt # Extract from PDF pdf2john encrypted.pdf > pdf_hash.txt

Defense Against Password Cracking

Protecting against password cracking attacks:

  • Strong Passwords: Use complex, long passwords
  • Salt Usage: Implement proper password salting
  • Slow Hashing: Use bcrypt, scrypt, or Argon2
  • Password Policies: Enforce minimum complexity
  • Multi-Factor Authentication: Add additional security layers
  • Regular Updates: Force periodic password changes
# Generate secure password openssl rand -base64 32 # Check password strength echo "password123" | john --stdin --stdout --rules | wc -l # Hash password securely echo "mypassword" | mkpasswd -m sha-512 -S randomsalt
