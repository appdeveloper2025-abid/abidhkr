Burp Suite Web Security Testing

What is Burp Suite?

Burp Suite is an integrated platform for performing security testing of web applications. It's designed to be used by hands-on testers to support the testing process.

Learning Objective: Master web application security testing using Burp Suite's comprehensive toolkit for finding and exploiting vulnerabilities.

Burp Suite Editions

Setup & Proxy Configuration

Installation

# Download from PortSwigger website https://portswigger.net/burp/communitydownload # Kali Linux installation sudo apt update sudo apt install burpsuite # Start Burp Suite burpsuite

Proxy Setup

# Default Burp Proxy Settings IP: 127.0.0.1 Port: 8080 # Browser Proxy Configuration (Firefox) 1. Open Firefox 2. Go to Settings → Network Settings 3. Select "Manual proxy configuration" 4. HTTP Proxy: 127.0.0.1, Port: 8080 5. Check "Use this proxy server for all protocols" # Install Burp CA Certificate 1. Browse to http://burp while proxy is running 2. Click "CA Certificate" to download 3. Import certificate in browser

Intercept Configuration

# Proxy Tab Settings 1. Go to Proxy → Intercept 2. Turn "Intercept is on/off" as needed 3. Configure intercept rules in Options 4. Set scope to target specific domains # Useful Intercept Rules - Intercept requests to specific URLs - Intercept responses with specific content - Skip common file types (images, CSS, JS)

Spider & Scanner

Web Crawler (Spider)

# Manual Spidering 1. Browse target application normally 2. Burp automatically maps the application 3. Review Site Map in Target tab 4. Right-click → Spider this host # Spider Configuration - Set crawl limits - Configure authentication - Exclude specific paths - Set crawl optimization

Vulnerability Scanner

# Active Scanning (Pro version) 1. Right-click on target in Site Map 2. Select "Actively scan this host" 3. Configure scan settings 4. Monitor scan progress # Passive Scanning - Automatically analyzes all traffic - Identifies potential issues - No additional requests sent - Available in Community Edition

Intruder Attack Tool

Attack Types

Sniper

Single payload set, one position at a time

# Use case: Username enumeration POST /login username=§admin§&password=test
Battering Ram

Single payload set, all positions simultaneously

# Use case: Same value in multiple fields username=§admin§&password=§admin§

Payload Types

# Simple List - Load from file - Paste from clipboard - Manual entry # Brute Forcer - Character sets - Min/max length - Case variations # Numbers - Sequential numbers - Random numbers - Hex numbers # Runtime File - Read from file during attack - Useful for large wordlists

Common Attack Scenarios

# Password Brute Force 1. Capture login request 2. Send to Intruder 3. Mark password field as payload position 4. Load password wordlist 5. Start attack and analyze responses # SQL Injection Testing 1. Identify injection points 2. Load SQL injection payloads 3. Analyze response differences 4. Look for error messages or timing differences # Directory Enumeration 1. Capture directory request 2. Mark directory name as payload position 3. Load directory wordlist (dirb, dirbuster) 4. Look for 200 response codes

Repeater Manual Testing

Using Repeater

# Send Request to Repeater 1. Right-click on any request 2. Select "Send to Repeater" 3. Modify request in Repeater tab 4. Click "Go" to send modified request 5. Analyze response # Repeater Features - Syntax highlighting - Request/response history - Multiple tabs for different requests - Raw/pretty/hex views

Manual Testing Techniques

# SQL Injection Testing Original: id=1 Test: id=1' Test: id=1' OR '1'='1 Test: id=1' UNION SELECT 1,2,3-- # XSS Testing Original: search=test Test: search= Test: search="> Test: search=javascript:alert(1) # Authentication Bypass Original: admin=false Test: admin=true Test: role=admin Test: Remove authentication headers

Response Analysis

# Look for: - Status code changes (200, 302, 403, 500) - Response length differences - Error messages in response - Timing differences - New headers or cookies - Content changes # Compare Responses 1. Use "Response" dropdown 2. Select "Compare responses" 3. Analyze differences between requests
