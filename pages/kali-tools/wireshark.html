Wireshark Network Analysis

What is Wireshark?

Wireshark is the world's foremost network protocol analyzer. It lets you capture and interactively browse the traffic running on a computer network.

Learning Objective: Master network packet analysis, protocol inspection, and traffic monitoring using Wireshark for cybersecurity investigations.

Core Theory

Wireshark is like having X-ray vision for networks. Just as doctors use X-rays to see inside the human body, security professionals use Wireshark to see inside network communications. Every packet tells a story about what's happening on the network.

Packet Analysis Fundamentals

Understanding packet analysis requires knowledge of network communication principles:

  1. Promiscuous Mode: Network adapter captures all packets, not just those destined for it
  2. Protocol Stack: Each layer adds headers with specific information
  3. Timing Analysis: Packet timestamps reveal communication patterns
  4. Flow Reconstruction: Reassembling conversations from individual packets

Key Features

  • Live Capture: Real-time packet capture
  • Protocol Support: 1000+ protocols
  • Deep Inspection: Packet-level analysis
  • Filtering: Advanced display filters
  • Statistics: Network statistics
  • Export: Multiple file formats
  • Coloring Rules: Visual packet identification
  • Follow Streams: TCP/UDP stream analysis

Security Analysis Applications

How Security Professionals Use Wireshark
  • Incident Response: Analyzing network traffic during security incidents
  • Malware Analysis: Understanding malicious network behavior
  • Penetration Testing: Verifying attack success and data flow
  • Forensics: Reconstructing network-based attacks
  • Compliance: Monitoring for policy violations

Wireshark Interface

Main Components

Packet List Pane

Shows captured packets in chronological order

  • Packet number
  • Timestamp
  • Source/Destination
  • Protocol
  • Length
  • Info
Packet Details Pane

Shows protocol layers of selected packet

  • Frame information
  • Ethernet header
  • IP header
  • TCP/UDP header
  • Application data

Capture Techniques

Promiscuous Mode

Capture all traffic on network segment

# Enable in capture options Interface → Promiscuous: ON
Monitor Mode

Capture wireless traffic (WiFi)

# Enable monitor mode sudo airmon-ng start wlan0 wireshark -i wlan0mon
Remote Capture

Capture from remote systems

# SSH tunnel ssh -L 2222:localhost:22 user@remote wireshark -k -i TCP@localhost:2222

Security Analysis

# Detect Port Scanning tcp.flags.syn==1 and tcp.flags.ack==0 # Find Failed Login Attempts http.request.method=="POST" and http contains "login" # Detect Suspicious DNS Queries dns.qry.name contains "suspicious-domain.com" # Find Large Data Transfers frame.len > 1500 # Detect Malware Communication ip.addr == known_malware_ip

Display Filters

Basic Filters

# IP Address Filters ip.addr == 192.168.1.100 # Specific IP ip.src == 192.168.1.100 # Source IP ip.dst == 192.168.1.100 # Destination IP ip.addr == 192.168.1.0/24 # Network range # Port Filters tcp.port == 80 # TCP port 80 udp.port == 53 # UDP port 53 tcp.srcport == 443 # Source port 443 tcp.dstport == 22 # Destination port 22 # Protocol Filters http # HTTP traffic https # HTTPS traffic dns # DNS traffic icmp # ICMP traffic arp # ARP traffic

Advanced Filters

# Logical Operators ip.addr == 192.168.1.100 and tcp.port == 80 ip.addr == 192.168.1.100 or ip.addr == 192.168.1.101 not arp and not icmp # String Matching http contains "password" tcp contains "malware" frame contains "suspicious" # Time-based Filters frame.time >= "2024-01-01 00:00:00" frame.time_relative > 10 # Size-based Filters frame.len > 1000 tcp.len == 0 # TCP ACK packets

Security-Focused Filters

# Detect Suspicious Activity tcp.flags.reset == 1 # TCP resets tcp.analysis.retransmission # Retransmissions tcp.analysis.duplicate_ack # Duplicate ACKs # Web Security http.request.method == "POST" # POST requests http.response.code >= 400 # HTTP errors ssl.handshake.type == 1 # SSL handshakes # Network Reconnaissance icmp.type == 8 # Ping requests tcp.flags.syn == 1 and tcp.flags.ack == 0 # SYN scans dns.qry.type == 1 # A record queries

Filter Macros

# Create custom macros 1. Go to View → Coloring Rules 2. Add new rule with filter expression 3. Choose color for matching packets # Common Security Macros ${web_traffic} = tcp.port == 80 or tcp.port == 443 ${suspicious_ports} = tcp.port == 1337 or tcp.port == 31337 ${internal_network} = ip.addr == 192.168.0.0/16
