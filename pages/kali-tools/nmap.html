    Introduction to Nmap

    Nmap (Network Mapper) is a free and open-source network discovery and security auditing tool. It's one of the most essential tools in a penetration tester's arsenal.

    Core Theory

    Nmap works by sending specially crafted packets to target systems and analyzing the responses. Think of it as "knocking on doors" of a building to see which rooms are occupied and what's inside each room.

    How Network Scanning Works

    Network scanning is based on fundamental networking principles:

    1. TCP Three-Way Handshake: SYN → SYN-ACK → ACK sequence for connection establishment
    2. Port States: Open (service listening), Closed (no service), Filtered (firewall blocking)
    3. ICMP Protocol: Used for host discovery and network diagnostics
    4. Service Fingerprinting: Analyzing service banners and responses to identify software

    Scanning Theory & Methodology

    Stealth Considerations

    • SYN Scan: Half-open scan, doesn't complete connection
    • Timing: Slower scans avoid detection
    • Fragmentation: Split packets to evade firewalls
    • Decoy Scanning: Hide among fake source IPs
    • Vulnerability Scanning: NSE scripts
    • Firewall Evasion: Bypass security measures
    • Output Formats: Multiple report formats
    • Timing Control: Stealth scanning options

    Installation

    Kali Linux (Pre-installed)

    nmap --version

    Ubuntu/Debian

    sudo apt update sudo apt install nmap

    Windows

    Download from nmap.org and run the installer.

    macOS

    brew install nmap

    Basic Scanning Techniques

    1. Basic Host Discovery

    # Ping scan - discover live hosts nmap -sn 192.168.1.0/24 # Scan specific IP nmap 192.168.1.1 # Scan multiple IPs nmap 192.168.1.1-10

    2. Port Scanning

    # TCP SYN scan (default) nmap -sS target_ip # TCP connect scan nmap -sT target_ip # UDP scan nmap -sU target_ip # Scan specific ports nmap -p 80,443,22 target_ip # Scan port ranges nmap -p 1-1000 target_ip

    3. Service Version Detection

    # Service version detection nmap -sV target_ip # Aggressive service detection nmap -sV --version-intensity 9 target_ip # Light service detection nmap -sV --version-intensity 0 target_ip
    Legal Notice: Only scan networks you own or have explicit permission to test. Unauthorized scanning is illegal.

    Advanced Scanning Techniques

    1. Operating System Detection

    # OS fingerprinting nmap -O target_ip # Aggressive OS detection nmap -O --osscan-guess target_ip # OS detection with version scanning nmap -O -sV target_ip

    2. Comprehensive Scanning

    # Aggressive scan (OS, version, script, traceroute) nmap -A target_ip # Comprehensive scan with all TCP ports nmap -A -p- target_ip # Fast scan (top 100 ports) nmap -F target_ip

    3. Timing Templates

    # Timing templates (0-5) -T0 # Paranoid (very slow, IDS evasion) -T1 # Sneaky (slow, IDS evasion) -T2 # Polite (slower, less bandwidth) -T3 # Normal (default timing) -T4 # Aggressive (faster, reliable networks) -T5 # Insane (very fast, may miss results)
    # Examples nmap -T4 -A target_ip # Fast aggressive scan nmap -T1 -sS target_ip # Slow stealth scan

    Nmap Scripting Engine (NSE)

    Script Categories

    • auth: Authentication bypass
    • broadcast: Network broadcast discovery
    • brute: Brute force attacks
    • default: Default safe scripts
    • discovery: Network discovery
    • dos: Denial of service
    • exploit: Exploit vulnerabilities
    • fuzzer: Fuzzing attacks
    • intrusive: Intrusive scripts
    • vuln: Vulnerability detection

    Common NSE Scripts

    # Run default scripts nmap -sC target_ip # Run specific script nmap --script http-title target_ip # Run script category nmap --script vuln target_ip # Multiple scripts nmap --script "http-* and not http-brute" target_ip # Vulnerability scanning nmap --script vuln,safe target_ip

    Popular Scripts Examples

    # HTTP information gathering nmap --script http-enum target_ip nmap --script http-headers target_ip nmap --script http-methods target_ip # SMB enumeration nmap --script smb-enum-shares target_ip nmap --script smb-os-discovery target_ip # DNS enumeration nmap --script dns-brute target_domain # SSL/TLS testing nmap --script ssl-enum-ciphers -p 443 target_ip

    Stealth and Evasion Techniques

    1. Fragmentation

    # Fragment packets nmap -f target_ip # Specify fragment size nmap --mtu 16 target_ip

    2. Decoy Scanning

    # Use decoy IPs nmap -D 192.168.1.100,192.168.1.101,ME target_ip # Random decoys nmap -D RND:10 target_ip

    3. Source Port Spoofing

    # Spoof source port nmap --source-port 53 target_ip nmap -g 80 target_ip

    4. Idle Scan (Zombie Scan)

    # Use zombie host for scanning nmap -sI zombie_ip target_ip
    Pro Tip: Combine multiple evasion techniques for maximum stealth, but remember this may reduce scan accuracy.

    Practical Scanning Scenarios

    Scenario 1: Network Discovery

    # Discover live hosts in network nmap -sn 192.168.1.0/24 # Save results to file nmap -sn 192.168.1.0/24 > live_hosts.txt

    Scenario 2: Web Server Assessment

    # Comprehensive web server scan nmap -sS -sV -O --script http-enum,http-headers,http-methods,http-title -p 80,443,8080,8443 target_ip # SSL/TLS security assessment nmap --script ssl-cert,ssl-enum-ciphers,ssl-heartbleed -p 443 target_ip

    Scenario 3: Network Infrastructure Scan

    # Scan for network devices nmap -sS -O --script snmp-info,snmp-sysdescr -p 161 network_range # Router/Switch discovery nmap --script broadcast-dhcp-discover

    Output Formats

    # Normal output nmap target_ip # XML output nmap -oX scan_results.xml target_ip # Grepable output nmap -oG scan_results.gnmap target_ip # All formats nmap -oA scan_results target_ip

    Reading Nmap Results

    # Port States open # Port is accepting connections closed # Port is accessible but no service listening filtered # Port is blocked by firewall/filter unfiltered # Port is accessible but state cannot be determined open|filtered # Port is either open or filtered # Service Detection 22/tcp open ssh OpenSSH 7.4 (protocol 2.0) 80/tcp open http Apache httpd 2.4.6 443/tcp open https Apache httpd 2.4.6 ((CentOS))
