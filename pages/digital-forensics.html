Digital Forensics & Incident Response

Introduction to Digital Forensics

Digital forensics is the process of uncovering and interpreting electronic data for use in a court of law. It combines computer science and investigative procedures to recover digital evidence.

Core Theory

Think of digital forensics as being a detective in the digital world. Just as traditional detectives collect fingerprints and DNA evidence, digital forensics experts collect digital "fingerprints" - file timestamps, network logs, deleted files, and system artifacts that tell the story of what happened on a computer system.

Types of Digital Forensics

By Device Type

  • Computer Forensics: Desktop and laptop analysis
  • Mobile Forensics: Smartphones and tablets
  • Network Forensics: Network traffic analysis
  • Cloud Forensics: Cloud-based evidence

By Application

  • Criminal Investigations
  • Corporate Security Incidents
  • Civil Litigation
  • Regulatory Compliance

Digital Forensics Process

A systematic approach ensures evidence integrity and legal admissibility.

Forensics Methodology
  1. Identification: Recognize potential evidence sources
  2. Preservation: Secure and protect evidence from alteration
  3. Collection: Acquire evidence using forensically sound methods
  4. Examination: Process and extract relevant information
  5. Analysis: Interpret findings and draw conclusions
  6. Presentation: Report findings clearly and accurately

Chain of Custody

Maintaining an unbroken chain of custody is crucial for legal proceedings:

  • Documentation: Record every person who handles evidence
  • Timestamps: Log when evidence was accessed or transferred
  • Security: Ensure evidence is stored securely
  • Integrity: Use cryptographic hashes to verify evidence hasn't been altered

Evidence Acquisition

Proper evidence acquisition ensures data integrity and legal admissibility.

Live vs. Dead Acquisition

Live Acquisition

  • Volatile Memory: RAM contents
  • Running Processes: Active system state
  • Network Connections: Current network activity
  • Encrypted Volumes: Mounted encrypted drives

Dead Acquisition

  • Complete Disk Image: Bit-for-bit copy
  • File System Analysis: Deleted file recovery
  • Metadata Preservation: File timestamps
  • Unallocated Space: Hidden data recovery
# Creating forensic disk image with dd dd if=/dev/sda of=/mnt/evidence/disk_image.dd bs=4096 conv=noerror,sync # Creating image with dcfldd (enhanced dd) dcfldd if=/dev/sda of=/mnt/evidence/disk_image.dd hash=sha256 hashlog=/mnt/evidence/hash.log # Memory acquisition with LiME insmod lime.ko "path=/mnt/evidence/memory.lime format=lime" # Volatility memory analysis volatility -f memory.lime --profile=LinuxUbuntu1604x64 linux_pslist

Analysis Techniques

Various techniques are used to extract meaningful information from digital evidence.

File System Analysis

Timeline Analysis

Creating timelines helps understand the sequence of events:

# Create timeline with log2timeline/plaso log2timeline.py --storage-file timeline.plaso disk_image.dd # Convert to CSV format psort.py -o l2tcsv -w timeline.csv timeline.plaso # Timeline analysis with mactime fls -r -m / disk_image.dd > bodyfile.txt mactime -b bodyfile.txt -d > timeline.txt

Incident Response

Incident response is the organized approach to addressing and managing security breaches or cyberattacks.

NIST Incident Response Framework

IR Lifecycle
  1. Preparation: Establish IR capabilities and procedures
  2. Detection & Analysis: Identify and investigate incidents
  3. Containment, Eradication & Recovery: Stop the attack and restore systems
  4. Post-Incident Activity: Learn from the incident and improve

Incident Classification

By Severity

  • Low: Minimal business impact
  • Medium: Moderate business impact
  • High: Significant business impact
  • Critical: Severe business impact

By Type

  • Malware Infection
  • Data Breach
  • Denial of Service
  • Insider Threat
