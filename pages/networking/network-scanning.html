Network Scanning Techniques

What is Network Scanning?

Network scanning is the process of identifying active hosts, open ports, and services running on a network. It's a fundamental reconnaissance technique used by both ethical hackers and malicious attackers.

Learning Objective: Master various network scanning techniques, understand their purposes, and learn how to defend against malicious scanning attempts.

Why Network Scanning Matters

For Security Professionals
  • Identify network assets and inventory
  • Discover unauthorized devices
  • Assess network security posture
  • Validate firewall configurations
  • Monitor for rogue services
For Attackers (Know Your Enemy)
  • Map network topology
  • Identify potential targets
  • Find vulnerable services
  • Plan attack vectors
  • Gather intelligence for exploitation

Types of Network Scans

1. Host Discovery Scans

Determine which hosts are alive on the network

# ICMP Echo (Ping) Scan nmap -sn 192.168.1.0/24 # TCP SYN Ping nmap -PS80,443,22 192.168.1.0/24 # UDP Ping nmap -PU53,67,68 192.168.1.0/24 # ARP Ping (Local network) nmap -PR 192.168.1.0/24

2. Port Scanning

Identify open ports and services on target hosts

TCP Scans
# TCP SYN Scan (Stealth) nmap -sS target # TCP Connect Scan nmap -sT target # TCP ACK Scan nmap -sA target # TCP Window Scan nmap -sW target
UDP Scans
# UDP Scan nmap -sU target # UDP with version detection nmap -sUV target # Fast UDP scan (top ports) nmap -sU --top-ports 100 target

3. Service and Version Detection

# Service version detection nmap -sV target # Aggressive service detection nmap -sV --version-intensity 9 target # OS fingerprinting nmap -O target # Comprehensive scan nmap -A target

Host Discovery Techniques

ICMP-Based Discovery

# Traditional ping sweep for i in {1..254}; do ping -c 1 -W 1 192.168.1.$i &> /dev/null && echo "192.168.1.$i is alive" done # Using fping for faster scanning fping -a -g 192.168.1.0/24 2>/dev/null # Nmap ping sweep nmap -sn 192.168.1.0/24

TCP-Based Discovery

# TCP SYN ping to common ports nmap -PS21,22,23,25,53,80,110,443,993,995 192.168.1.0/24 # TCP ACK ping nmap -PA80,443 192.168.1.0/24 # Specific port combinations nmap -PS80 -PA3389 192.168.1.0/24

ARP Discovery (Local Networks)

# ARP scan with nmap nmap -PR 192.168.1.0/24 # Using arp-scan arp-scan -l arp-scan 192.168.1.0/24 # Manual ARP table check arp -a
Pro Tip: ARP scanning is the most reliable method for discovering hosts on local networks since it operates at Layer 2 and bypasses most firewalls.

Advanced Port Scanning

TCP Scan Types Explained

SYN Scan (-sS)

How it works:

1. Send SYN packet 2. If port open: receive SYN-ACK 3. Send RST to close connection 4. If port closed: receive RST

Advantages: Stealthy, fast, doesn't complete connections

Connect Scan (-sT)

How it works:

1. Complete TCP 3-way handshake 2. Establish full connection 3. Close connection properly

Advantages: Works without root privileges, accurate results

UDP Scanning Challenges

UDP Scanning Difficulties
  • Slow: UDP is connectionless, responses are optional
  • Rate Limited: ICMP responses are often rate-limited
  • Firewalls: Often filtered without clear indication
  • False Positives: Difficult to distinguish open vs. filtered
# Effective UDP scanning techniques nmap -sU --top-ports 100 target nmap -sU -sV --version-intensity 0 target nmap -sU --max-retries 1 target

Port Range Strategies

# Scan specific ports nmap -p 80,443,22,21,25 target # Scan port ranges nmap -p 1-1000 target nmap -p 1000-2000 target # Scan all ports nmap -p- target # Top ports only (faster) nmap --top-ports 100 target nmap --top-ports 1000 target

Stealth Scanning Techniques

Timing and Performance

# Timing templates (0-5) nmap -T0 target # Paranoid (very slow) nmap -T1 target # Sneaky (slow) nmap -T2 target # Polite (slower) nmap -T3 target # Normal (default) nmap -T4 target # Aggressive (fast) nmap -T5 target # Insane (very fast) # Custom timing nmap --scan-delay 5s target nmap --max-rate 100 target

Fragmentation and Evasion

# Fragment packets nmap -f target # Specify MTU size nmap --mtu 16 target # Decoy scanning nmap -D 192.168.1.100,192.168.1.101,ME target # Source port spoofing nmap --source-port 53 target nmap -g 80 target # Randomize target order nmap --randomize-hosts target1 target2 target3

Advanced Evasion

# Idle scan (zombie scan) nmap -sI zombie_host target # FTP bounce scan nmap -b ftp_server target # Custom TCP flags nmap --scanflags URGACKPSHRSTSYNFIN target # Bad checksum (test IDS) nmap --badsum target

Defending Against Network Scans

Detection Methods

Network-Based Detection
  • IDS/IPS: Snort, Suricata
  • Network Monitoring: Wireshark, tcpdump
  • Flow Analysis: NetFlow, sFlow
  • Honeypots: Detect scanning attempts
Host-Based Detection
  • Log Analysis: System and application logs
  • Port Scan Detectors: PortSentry, PSAD
  • HIDS: OSSEC, Samhain
  • Behavioral Analysis: Unusual connection patterns

Prevention Strategies

# Firewall rules to limit scanning iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m recent --set iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m recent --update --seconds 60 --hitcount 4 -j DROP # Rate limiting with iptables iptables -A INPUT -p tcp --dport 80 -m limit --limit 25/minute --limit-burst 100 -j ACCEPT # Block common scan patterns iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP iptables -A INPUT -p tcp --tcp-flags ALL ALL -j DROP

Response Actions

Incident Response for Port Scans
  1. Log and Document: Record source IP, time, ports scanned
  2. Analyze Pattern: Determine scan type and intent
  3. Block Source: Temporarily or permanently block scanning IP
  4. Investigate: Check for successful compromises
  5. Harden Systems: Close unnecessary ports and services
  6. Monitor: Increase monitoring for follow-up attacks

Practical Defense Example
