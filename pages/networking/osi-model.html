OSI Model

Understanding the 7-layer network communication model

Introduction to OSI Model

The Open Systems Interconnection (OSI) model is a conceptual framework that standardizes the functions of a telecommunication or computing system into seven abstraction layers. Each layer serves a specific function and communicates with the layers directly above and below it.

Layer Name Function Examples
7 Application User interface HTTP, FTP, SMTP
6 Presentation Data formatting SSL/TLS, JPEG, ASCII
5 Session Session management NetBIOS, RPC
4 Transport End-to-end delivery TCP, UDP
3 Network Routing IP, ICMP, OSPF
2 Data Link Node-to-node delivery Ethernet, WiFi, PPP
1 Physical Physical transmission Cables, Radio, Fiber

Layer 7: Application Layer

The Application Layer provides network services directly to end-users and applications.

Functions

  • User interface for network services
  • Application-specific protocols
  • Data exchange between applications
  • Authentication and authorization
# Common Application Layer protocols HTTP/HTTPS - Web browsing FTP/SFTP - File transfer SMTP/POP3/IMAP - Email DNS - Domain name resolution DHCP - IP address assignment SSH - Secure shell access Telnet - Remote terminal access

Layer 6: Presentation Layer

The Presentation Layer handles data formatting, encryption, and compression.

Functions

  • Data encryption and decryption
  • Data compression and decompression
  • Character encoding (ASCII, Unicode)
  • Data format conversion
# Presentation Layer examples SSL/TLS - Encryption protocols JPEG, GIF, PNG - Image formats MPEG, AVI - Video formats ASCII, EBCDIC - Character encoding ZIP, RAR - Compression formats

Layer 5: Session Layer

The Session Layer manages sessions between applications.

Functions

  • Session establishment and termination
  • Session checkpointing and recovery
  • Dialog control (full-duplex, half-duplex)
  • Synchronization
# Session Layer protocols NetBIOS - Network Basic Input/Output System RPC - Remote Procedure Call SQL sessions - Database connections PPTP - Point-to-Point Tunneling Protocol

Layer 4: Transport Layer

The Transport Layer provides reliable data transfer between end systems.

Functions

  • Segmentation and reassembly
  • Error detection and correction
  • Flow control
  • Port addressing
# Transport Layer protocols TCP (Transmission Control Protocol) - Connection-oriented - Reliable delivery - Error checking - Flow control UDP (User Datagram Protocol) - Connectionless - Fast transmission - No error checking - Best effort delivery # Port examples HTTP: 80, HTTPS: 443 FTP: 21, SSH: 22 Telnet: 23, SMTP: 25 DNS: 53, DHCP: 67/68

Layer 3: Network Layer

The Network Layer handles routing and logical addressing.

Functions

  • Logical addressing (IP addresses)
  • Routing between networks
  • Path determination
  • Packet forwarding
# Network Layer protocols IP (Internet Protocol) - IPv4: 32-bit addresses - IPv6: 128-bit addresses ICMP (Internet Control Message Protocol) - Error reporting - Network diagnostics Routing Protocols: - OSPF (Open Shortest Path First) - BGP (Border Gateway Protocol) - RIP (Routing Information Protocol) # Network Layer tools ping - Test connectivity traceroute - Trace packet path netstat - Network statistics route - View/modify routing table

Layer 2: Data Link Layer

The Data Link Layer provides node-to-node delivery and error detection.

Functions

  • Physical addressing (MAC addresses)
  • Frame synchronization
  • Error detection
  • Flow control
# Data Link Layer protocols Ethernet - Wired LAN WiFi (802.11) - Wireless LAN PPP - Point-to-Point Protocol Frame Relay - WAN protocol ATM - Asynchronous Transfer Mode # MAC Address format 48-bit address: 00:1A:2B:3C:4D:5E First 24 bits: Vendor identifier (OUI) Last 24 bits: Device identifier # Data Link Layer tools arp - View/modify ARP table ifconfig - Configure network interface iwconfig - Configure wireless interface

Layer 1: Physical Layer

The Physical Layer handles the physical transmission of data.

Functions

  • Bit transmission
  • Physical topology
  • Electrical specifications
  • Mechanical specifications
# Physical Layer components Cables: - Ethernet (Cat5e, Cat6, Cat6a) - Fiber optic (Single-mode, Multi-mode) - Coaxial cables Wireless: - Radio frequencies - Infrared - Microwave Devices: - Hubs - Repeaters - Network interface cards - Transceivers

Security Implications by Layer

Understanding security vulnerabilities at each OSI layer:

Upper Layers (5-7)

  • Application: SQL injection, XSS, CSRF
  • Presentation: Encryption weaknesses
  • Session: Session hijacking, fixation

Lower Layers (1-4)

  • Transport: Port scanning, DoS attacks
  • Network: IP spoofing, routing attacks
  • Data Link: MAC spoofing, ARP poisoning
  • Physical: Cable tapping, jamming
# Security tools by layer Application Layer: - Burp Suite, OWASP ZAP - Nikto, SQLmap Network Layer: - Nmap, Nessus - Wireshark, tcpdump Data Link Layer: - Ettercap, Bettercap - Aircrack-ng (wireless) # Defense strategies Layer 7: Input validation, WAF Layer 6: Strong encryption (AES, RSA) Layer 5: Session management Layer 4: Firewalls, IDS/IPS Layer 3: Router ACLs, VPNs Layer 2: Port security, VLANs Layer 1: Physical security
