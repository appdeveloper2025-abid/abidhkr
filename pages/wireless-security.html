Wireless Security & WiFi Hacking

Introduction to Wireless Security

Wireless networks have become ubiquitous in modern computing, but they present unique security challenges. Understanding wireless security is crucial for protecting against unauthorized access and data interception.

Core Theory

Think of wireless networks like radio broadcasts - anyone with the right equipment can potentially "tune in" to your signal. Unlike wired networks where physical access is required, wireless signals travel through the air and can be intercepted from a distance, making security protocols essential.

Wireless Security Challenges

Physical Layer Issues

  • Signal Propagation: Radio waves travel beyond intended boundaries
  • Interference: Other devices can disrupt communications
  • Eavesdropping: Passive monitoring is difficult to detect
  • Range Limitations: Signal strength affects security

Protocol Vulnerabilities

  • Weak Encryption: Outdated security protocols
  • Authentication Flaws: Weak credential verification
  • Key Management: Poor key distribution and rotation
  • Implementation Bugs: Software vulnerabilities

WiFi Fundamentals

Understanding how WiFi works is essential for comprehending its security implications.

IEEE 802.11 Standards

WiFi Evolution
802.11 (1997)
Original standard, 2 Mbps, 2.4 GHz
802.11b (1999)
11 Mbps, 2.4 GHz, widespread adoption
802.11g (2003)
54 Mbps, 2.4 GHz, backward compatible
802.11n (2009)
600 Mbps, 2.4/5 GHz, MIMO technology
802.11ac (2013)
6.93 Gbps, 5 GHz, improved MIMO
802.11ax (2019)
9.6 Gbps, WiFi 6, enhanced efficiency

WiFi Architecture

Infrastructure Mode

  • Access Point (AP): Central communication hub
  • Basic Service Set (BSS): AP and associated stations
  • Extended Service Set (ESS): Multiple connected APs
  • Distribution System: Backend network infrastructure

Ad-Hoc Mode

  • Independent BSS (IBSS): Direct device communication
  • Peer-to-Peer: No central access point
  • Limited Range: Direct radio communication only
  • Security Challenges: Difficult to manage

Wireless Security Protocols

Various security protocols have been developed to protect wireless communications, each with different strengths and vulnerabilities.

WEP (Wired Equivalent Privacy)

Deprecated Protocol

WEP is fundamentally broken and should never be used. It's included here for educational purposes only.

# WEP Vulnerabilities - Static encryption keys - Weak IV (Initialization Vector) implementation - RC4 stream cipher weaknesses - No key management - Easily cracked with tools like aircrack-ng # WEP Cracking Example airmon-ng start wlan0 airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon aireplay-ng -1 0 -a AA:BB:CC:DD:EE:FF wlan0mon aircrack-ng capture-01.cap

WPA/WPA2 (WiFi Protected Access)

WPA Improvements

  • TKIP: Temporal Key Integrity Protocol
  • Dynamic Keys: Per-packet key mixing
  • Message Integrity: Michael MIC
  • Replay Protection: Sequence counters

WPA2 Enhancements

  • AES-CCMP: Advanced Encryption Standard
  • Stronger Encryption: 128-bit AES
  • Improved Integrity: CCMP authentication
  • Enterprise Mode: 802.1X authentication

WPA3 (Latest Standard)

WPA3 Security Features
  • SAE (Simultaneous Authentication of Equals): Replaces PSK
  • Forward Secrecy: Past sessions remain secure
  • Brute Force Protection: Offline dictionary attack resistance
  • Enhanced Open: Opportunistic Wireless Encryption (OWE)
  • 192-bit Security: Government and enterprise grade

Wireless Attack Techniques

Understanding attack methods helps in developing effective defenses and conducting security assessments.

Passive Attacks

# Monitor Mode Setup airmon-ng check kill airmon-ng start wlan0 # Passive Network Discovery airodump-ng wlan0mon # Capture Handshakes airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w handshake wlan0mon # Analyze Captured Traffic wireshark handshake-01.cap

Active Attacks

Deauthentication Attacks

  • Purpose: Force clients to reconnect
  • Method: Send deauth frames
  • Goal: Capture handshakes
  • Impact: Denial of service

Evil Twin Attacks

  • Concept: Fake access point
  • Method: Mimic legitimate AP
  • Goal: Intercept communications
  • Tools: hostapd, dnsmasq

Wireless Defense Strategies

Implementing multiple layers of security helps protect wireless networks from various attack vectors.

Network Configuration

Basic Security

  • WPA3: Use latest security protocol
  • Strong Passwords: Complex PSK/passphrases
  • SSID Management: Non-descriptive network names
  • Guest Networks: Separate untrusted devices

Advanced Security

  • 802.1X: Enterprise authentication
  • MAC Filtering: Device access control
  • VPN: Additional encryption layer
  • Network Segmentation: VLAN isolation

Monitoring and Detection

# Wireless Intrusion Detection # Monitor for rogue access points airodump-ng --write rogue-scan wlan0mon # Detect deauthentication attacks tcpdump -i wlan0mon -s 65535 -w deauth-monitor.pcap # Analyze wireless traffic patterns tshark -r capture.pcap -Y "wlan.fc.type_subtype == 12" -T fields -e wlan.sa
